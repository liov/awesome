192.168.x.x、172.16.x.x - 172.31.x.x 和 10.x.x.x 都是 IPv4 私有地址（Private IP Addresses），由 IANA（互联网号码分配机构）在 RFC 1918 标准中定义。
它们在互联网上不可路由（即公网路由器会直接丢弃这些地址的数据包），只能在局域网内部使用。对于 Docker、家庭网络或企业内网来说，它们在功能上没有本质区别，主要区别在于地址空间大小、历史习惯和适用场景。
1. 核心区别对比表
   表格
   特性	A 类：10.x.x.x	B 类：172.16.x.x - 172.31.x.x	C 类：192.168.x.x
   完整范围	10.0.0.0 - 10.255.255.255	172.16.0.0 - 172.31.255.255	192.168.0.0 - 192.168.255.255
   默认掩码	/8 (255.0.0.0)	/12 (255.240.0.0)	/16 (255.255.0.0)
   可用主机数	约 1677 万 (巨大)	约 104 万 (中等)	约 6.5 万 (较小)
   子网灵活性	极高，可随意划分子网	高，适合中型网络	低，通常用于小局域网
   常见用途	大型企业、云厂商内网、K8s集群	中型企业、Docker 默认网段	家庭路由器、小型办公室
   Docker 默认	否 (需配置)	是 (172.17.0.0/16 起)	否 (常与物理网冲突)
2. 详细解析
   🟢 10.0.0.0/8 (A 类私有地址)
   特点：地址空间最大，只有一个大的网段 10.0.0.0。
   优势：
   扩展性极强：适合超大型网络。你可以将其划分为成千上万个子网（例如 10.0.1.0/24, 10.0.2.0/24...），而不用担心不够用。
   结构简单：所有内部 IP 都以 10. 开头，易于识别和管理。
   场景：
   云计算厂商（AWS, 阿里云, 腾讯云）的 VPC 默认通常推荐或使用 10.0.0.0/8。
   Kubernetes (K8s) 集群的 Pod 网段通常首选 10.x.x.x。
   大型企业内网。
   Docker 建议：如果你需要创建大量容器网络，或者担心与其他网段冲突，手动将 Docker 配置为使用 10.x 是最安全的（因为家庭路由器很少用这个段）。
   🔵 172.16.0.0/12 (B 类私有地址)
   特点：包含 16 个连续的 B 类网段（从 172.16 到 172.31）。
   优势：
   折中方案：比 192.168 大，比 10 小。适合中型网络。
   Docker 的“默认领地”：Docker 安装后，默认创建的 bridge 网络就是 172.17.0.0/16，后续创建的网络依次递增 (172.18, 172.19...)。
   场景：
   Docker/K8s 内部网络。
   中型公司内网。
   风险：正如你遇到的情况，如果宿主机物理网络也使用了 172.18.x.x（某些云厂商或特定设备喜欢用这个段），就会发生冲突。
   🟠 192.168.0.0/16 (C 类私有地址)
   特点：包含 256 个 C 类网段（192.168.0 到 192.168.255）。每个标准子网（/24）只有 254 个可用 IP。
   优势：
   历史惯性：几乎所有家用无线路由器（TP-Link, Asus, 小米等）出厂默认网关都是 192.168.0.1 或 192.168.1.1。
   简单：适合不需要复杂子网划分的小环境。
   场景：
   家庭网络。
   小型办公室。
   风险：极易冲突。如果你在家里或公司内网运行 Docker，而物理网络也是 192.168.x.x，Docker 如果自动分配到相同的子网，就会导致网络瘫痪。
